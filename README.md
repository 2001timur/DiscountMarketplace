Сучасна веб-орієнтована платформа екосистеми електронної комерції **«Маркетплейс послуг зі знижками»**. Застосунок автоматизує процеси взаємодії між постачальниками послуг (бізнесом) та кінцевими споживачами через централізовану систему дисконтних купонів (аналог відомих сервісів «Покупон» та «SuperDeal»).

Проєкт розроблено в межах курсового проєкту з дотриманням об’єктно-орієнтованої парадигми, принципів SOLID та сучасних інженерних практик розробки програмного забезпечення.



## Ключові особливості та функціонал

*   **Багаторівневий інтерфейс:** Розподілений доступ для трьох основних рольових моделей із надійним захистом маршрутів через спеціалізований сервіс `AuthService`.
*   **Інтелектуальне проєктування:** Логічну структуру системи та її архітектурні межі деталізовано за допомогою ментальних карт (Mind Map) на етапі аналізу аналогів ринку України.
*   **Чиста архітектура:** Повне відокремлення бізнес-логіки від рівня представлення даних за допомогою архітектурного шаблону **Repository/Service** та механізму **Dependency Injection** (впровадження залежностей).
*   **Ефективна робота з даними:** Оптимізація вибірок, фільтрацій та маніпуляцій з колекціями завдяки широкому використанню технології LINQ та узагальнених колекцій `System.Collections.Generic`.
*   **Адаптивний дизайн:** Клієнтська частина побудована на компонентній технології Blazor з використанням фреймворку Bootstrap для коректного відображення на будь-яких типах пристроїв.

## Доменна модель (Domain Model)

В основі системи лежить стійка об'єктно-орієнтована модель предметної області, що складається з 5 ключових сутностей. Між моделями встановлено логічні зв’язки типу **1:N (один-до-багатьох)**:

1.  `User` — Облікові записи користувачів, профілі та ролі.
2.  `Business` — Інформація про компанії-партнери та реєстрацію їхніх філій.
3.  `ServiceCategory` — Ієрархія та класифікація послуг, що надаються.
4.  `Deal` — Акційні пропозиції, умови знижок, ліміти та терміни дії.
5.  `Coupon` — Індивідуальні цифрові купони, згенеровані клієнтами для отримання знижки.

> 💡 *Загальну та детальну діаграми класів було автоматично згенеровано за допомогою вбудованих інструментів середовища розробки Visual Studio 2022.*



## Рольова матриця системи

| Роль | Функціональні обов'язки та можливості |
| :--- | :--- |
| **Адміністратор** | Повний моніторинг працездатності системи, модерація контенту, CRUD-операції над користувачами та бізнесами. |
| **Власник бізнесу** | Реєстрація та редагування філій, створення нових акційних пропозицій (Deals), аналітика використання купонів. |
| **Користувач/Клієнт** | Навігація каталогом, гнучка фільтрація за категоріями, генерація індивідуальних купонів, доступ до особистого кабінету. |



## Тестування та якість коду

В межах проєкту реалізовано комплексний підхід до забезпечення якості програмного забезпечення (QA):

*   **Модульне тестування (Unit Testing):** У тестовому проєкті `DiscountMarketplace.Test` розроблено та успішно виконано **41 модульний тест**. Покрита критично важлива бізнес-логіка:
    *   Обчислення фінальних цін з урахуванням акційних знижок.
    *   Валідація та контроль лімітів доступних купонів.
    *   Перевірка термінів дії акцій та автоматичне блокування протермінованих пропозицій.
*   **Функціональне тестування:** Проведено ручну верифікацію системи за заздалегідь розробленими сценаріями (тест-кейсами) для кожної з трьох ролей, що підтвердило повну відповідність маркетплейсу вихідним технічним вимогам.
